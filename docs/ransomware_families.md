# 主流勒索软件家族分析:行为与特征

本文从**防御/检测视角**归纳主流勒索家族的行为模式与可观测特征,并映射到本工程的
**PMU 硬件特征**与多层信号。内容为公开威胁情报的归纳,用于指导检测,不含攻击实现细节。

## 一、家族速览

| 家族 | 活跃期 | 加密算法 | 运营模式 | 标志性行为 |
|------|--------|---------|---------|-----------|
| **WannaCry** | 2017 | RSA + AES | 蠕虫自传播 | 利用 EternalBlue(SMB)蠕虫式扩散;杀手开关域名 |
| **LockBit** | 2019– | AES + RSA(自研加载器) | RaaS | **速度极快**(多线程加密)、自传播、删卷影、关防护 |
| **Conti / Ryuk** | 2018–2022 | AES + RSA | RaaS / 定向 | 人工操盘横向移动、按文件大小**部分加密**提速 |
| **BlackCat/ALPHV** | 2021– | ChaCha20/AES | RaaS(Rust 编写) | 跨平台(Win/Linux/ESXi)、可配置化、双重勒索 |
| **REvil/Sodinokibi** | 2019–2021 | Salsa20 + RSA | RaaS | 供应链攻击(Kaseya)、白名单地区豁免 |
| **Maze** | 2019–2020 | ChaCha20 + RSA | RaaS | **首创"双重勒索"**(先窃数据再加密) |
| **RansomEXX** | 2020– | AES + RSA | 定向 | 有 **Linux/ARM 版**,打 ESXi/服务器 |
| **Hive** | 2021–2023 | 自定义 | RaaS | 打医疗、用 Go 编写、部分加密 |
| **EvilQuest** | 2020 | 自定义 | macOS | Mac 平台、兼带窃密/键盘记录 |

## 二、勒索软件通用攻击链(Kill Chain)与各阶段特征

无论家族,生命周期大体一致。下面标注**每阶段的可观测信号**及**信号层次**(硬件 PMU /
文件系统 / 进程行为 / 网络)。

### 阶段 1:初始入侵
- 行为:钓鱼附件、RDP 暴破、漏洞利用、供应链投毒。
- 信号:**网络/邮件**(异常登录、可疑附件执行),硬件层无特异性。

### 阶段 2:落地与持久化
- 行为:释放载荷、写注册表/计划任务/服务、提权。
- 信号:**进程行为**(可疑子进程、自启动项写入)、签名缺失。

### 阶段 3:防御削弱(勒索特有,强信号)
- 行为:**删除卷影副本(VSS)**、停止备份/数据库/安全服务、清日志、禁用恢复。
- 信号:**进程行为/命令**(`vssadmin delete shadows`、停服务)——这是**早于加密**的
  高价值预警信号。

### 阶段 4:横向移动(企业定向家族)
- 行为:凭据窃取、SMB/PsExec/WMI 扩散、扫描共享。
- 信号:**网络**(内网横向连接激增)、异常远程执行。

### 阶段 5:加密(核心,PMU 强信号)
- 行为:**遍历磁盘 → 读文件 → 加密 → 写密文 → 改扩展名/投勒索信**。
- 信号(**本工程关注**):
  - **加密指令暴增**:AES/ChaCha → 加密扩展(`crypto_spec`)、SIMD(`ase_spec`)占比↑;
  - **全盘流式遍历**:L1/L2/LLC 缓存未命中↑、访存/指令比↑;
  - **写突发**:store 占比↑、写带宽↑;
  - **突发节奏**:短时间脉冲式完成(区别于平稳的合法加密);
  - **文件熵**:明文(低熵)→ 密文(高熵≈随机);扩展名批量改写。

### 阶段 6:勒索与外泄
- 行为:投递勒索信、**双重勒索**(外泄数据威胁公开)。
- 信号:**网络**(大量数据外传)、桌面/文件出现勒索信。

## 三、加密策略差异(直接影响 PMU 信号强度)

不同家族为**提速**采用不同加密策略,这显著改变硬件指纹:

| 策略 | 代表 | 说明 | 对 PMU 信号的影响 |
|------|------|------|------------------|
| **全文件加密** | 早期家族 | 加密整个文件 | 加密指令 + 写带宽信号**最强**、最易检测 |
| **部分/间歇加密** | LockBit、Conti、Hive | 只加密文件头部或按块间隔加密 | 加密指令占比**被稀释**,信号变弱,更隐蔽 |
| **按大小分级** | Conti | 小文件全加密、大文件部分加密 | 信号介于两者之间 |
| **多线程并行** | LockBit、BlackCat | 多线程同时加密 | 单位时间强度**极高**、节奏陡峭,**易被节奏特征捕捉** |

> ⚠️ 检测启示:**间歇加密是规避 PMU"加密指令占比"信号的主要手段**——这也是本工程
> 强调**缓存遍历 + 节奏(差分)特征**而非单纯依赖 `crypto_ratio` 的原因:即便加密指令
> 被稀释,"全盘遍历 + 突发节奏"仍难以隐藏。

## 四、按可观测特征归类(检测特征矩阵)

| 特征类别 | 具体信号 | 信号层 | 本工程是否覆盖 |
|---------|---------|--------|--------------|
| 加密计算 | AES/ChaCha 指令、SIMD 占比 | **硬件 PMU** | ✅ `crypto_ratio`/`ase_ratio` |
| 内存遍历 | L1/L2/LLC 未命中、访存/指令比 | **硬件 PMU** | ✅ `l1d_miss_rate`/`llc_mpki` |
| IO 模式 | 读全盘 + 写突发、store 占比 | **硬件 PMU** + 文件 | ✅ `st_ratio`(PMU 侧) |
| 时序节奏 | 突发脉冲 vs 平稳 | **硬件 PMU**(差分) | ✅ `d_*` 差分特征 |
| 文件熵 | 明文→密文熵跃升 | 文件系统 | ❌ 需补充 |
| 文件操作 | 批量改名/改扩展名、批量删除 | 文件系统 | ❌ 需补充 |
| 备份破坏 | 删卷影副本、停备份服务 | 进程行为 | ❌ 需补充(强预警) |
| 横向移动 | 内网 SMB/PsExec 激增 | 网络 | ❌ 需补充 |
| 身份 | 进程未签名、非白名单路径 | 身份 | ❌ 需补充 |

**结论**:本工程的 PMU 方案覆盖**加密计算 + 内存遍历 + 写模式 + 时序节奏**四类硬件
特征,是**早期、轻量、难规避**的一层;但"删卷影、改扩展名、文件熵、横向移动"等**最具
区分力的勒索特有行为**在硬件层不可见,生产级检测必须多层融合。

## 五、勒索软件 vs 合法负载的鉴别要点

最易误报的合法负载(见 [dataset.md](dataset.md) 干扰项)与勒索的关键区别:

| 合法负载 | 与勒索的相似 | **关键区别(检测依据)** |
|---------|------------|----------------------|
| TLS/磁盘加密 | 加密指令高 | 不全盘遍历(缓存命中好),无写突发,有签名 |
| 压缩/转码 | SIMD/访存高 | 加密指令低,熵不呈"明→密"跃升 |
| **加密备份** | 加密 **+ 全盘遍历**都高 | **节奏平稳**(非突发)、有签名、走已知备份进程、不删卷影 |
| 杀毒全盘扫描 | 全盘遍历 | 只读不写密文,加密指令低 |

→ 单一特征都可能撞车,**唯有"加密 + 全盘遍历 + 突发节奏 + 破坏备份"的组合**才是勒索
的强签名。本工程在硬件层抓前三项,第四项需进程行为层补充。

## 六、给检测系统的设计建议(落地)

1. **PMU 层(本工程)**:实时、低开销地捕捉"加密 + 遍历 + 写 + 节奏",作早期初筛;
2. **文件层**:监控文件熵跃变、批量扩展名改写、短时大量文件被改;
3. **行为层**:对"删卷影/停备份服务"等强预警行为做规则告警(可早于加密触发);
4. **身份层**:白名单合法备份/加密/AV 进程,大幅降低 encbackup 类误报;
5. **融合**:以 PMU 初筛触发,叠加文件/行为/身份信号做二级确认,平衡召回与精确率。

> 单层都不足以兼顾"早、准、难规避";PMU 的价值在于**早且难规避**,需与其他层互补。
