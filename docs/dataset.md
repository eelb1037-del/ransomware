# 数据集说明:训练 / 验证 / 测试 与 良性、勒索构成

本文档说明合成 PMU 数据集的**整体构成**(良性各子类 + 勒索)与**三向切分**
(训练 / 验证 / 测试)。数据由 `src/data_gen.py` 生成,默认随机种子 42。

> ⚠️ 数据为**合成**(按勒索软件硬件指纹建模),用于验证流水线与特征设计,**非真实
> 检测率**。可用真实 `perf` 采集替换(列名对齐 13 个 PMU 计数器即可)。

## 一、原始数据总量

| 维度 | 数值 |
|------|------|
| 总行数(采样区间) | 7798 |
| 执行流(pid) | 1200 |
| 行级标签 | 良性 7081 / 勒索 717 |
| 执行流级标签 | 良性 1085 / 勒索 115 |
| 每行特征 | 13 个 PMU 计数器(cpu_cycles 专用 + 12 通用) |

每个执行流(pid)是一条按时间排列的 PMU 事件序列;每行 = 该执行流某个采样区间的
事件增量计数。勒索软件占执行流的约 **9.6%**(类不均衡,贴近真实——勒索是少数)。

## 二、良性数据构成(4 类真实负载干扰项)

良性进程**不是**单一平稳负载,而是按真实占比混入 4 类,刻意制造与勒索的混淆:

| 子类 | 占良性比例 | ~执行流数 | 行为画像 | 对应真实进程 |
|------|:---:|:---:|---------|------------|
| **normal** | ~55% | ~596 | 加密/压缩都低,访存平稳 | 浏览器、办公、后台服务 |
| **crypto** | ~18% | ~195 | crypto+SIMD 高,但**不全盘遍历**(LLC 低) | TLS/HTTPS、磁盘加密、加密通信 |
| **compress** | ~12% | ~134 | SIMD 高、访存中等,但 **crypto 低** | zip/gzip、视频/媒体编码 |
| **encbackup** | ~15% | ~161 | **加密 + 全盘遍历都高**(最难) | 加密备份、全盘加密初始化 |

设计意图:
- `crypto` 让"是否加密"不再是充分判据(良性也加密);
- `compress` 让"SIMD/访存高"不再充分(良性也压缩);
- **`encbackup` 是最难干扰项**:与勒索在"加密 + 全盘遍历"两个核心维度**完全重叠**,
  唯一区别是**节奏**——它是平稳长流,勒索是突发脉冲。逼模型用差分/方差(节奏)而非
  绝对水平来区分。

## 三、勒索数据构成

| 维度 | 说明 |
|------|------|
| 执行流数 | 115(占全部 ~9.6%) |
| 加密前 | 基线行为与良性相近(潜伏期,避免任务过于平凡) |
| 加密期 | 某时刻(onset)起进入"加密 + 全盘遍历",由 `ramp` 渐强驱动 |
| 关键指纹 | crypto/ase 占比↑、L1/L2/LLC 未命中↑、写回↑、且呈**突发上升**(与 encbackup 的平稳相区分) |

勒索软件 = **加密 + 全盘流式遍历 + 突发节奏**。前两者 encbackup 也有,**突发节奏**是
区分二者的关键,体现在 `d_*` 差分特征与窗口内方差上。

## 四、窗口化与三向切分

每个执行流按 `win=32 / stride=16` 切成滑动窗口;窗口标签 = 窗内任一时刻为恶意即为 1。

| 集合 | 窗口数 | 正例(勒索) | 正例率 | 执行流数 |
|------|:---:|:---:|:---:|:---:|
| **训练 train** | 770 | 74 | 9.6% | 768 |
| **验证 val** | 194 | 19 | 9.8% | 192 |
| **测试 test** | 241 | 23 | 9.5% | 240 |
| 合计 | 1205 | 116 | 9.6% | 1200 |

切分原则(`train.py: group_stratified_split`):
- **按 pid 分组**:同一执行流的所有窗口**只进同一个集合**,严防同源数据跨集合泄漏;
- **标签分层**:三个集合的勒索正例率保持一致(均 ~9.6%),避免分布偏移;
- 比例 ≈ 64 / 16 / 20。

各集合用途:
- **train**:拟合 XGBoost、LSTM,以及标准化器(`StandardScaler`/序列均值方差,仅在
  train 上 fit,防泄漏);
- **val**:LSTM 早停、XGBoost 早停、训练 stacking 元学习器、**选最优阈值**(最大化 F1);
- **test**:最终评估,所有指标(ROC-AUC / PR-AUC / F1 / 精确率 / 召回率)均在此报告。

## 五、复现

```bash
cd src
python3 data_gen.py                 # 生成数据集(种子 42)
python3 data_gen.py --n-proc 2000 --ransom-frac 0.15 --seed 7   # 自定义规模/比例
python3 train.py                    # 切分 + 训练 + 验证 + 测试
```

> 切分在 `train.py` 内部按固定种子完成,`data_gen.py` 与 `train.py` 同种子即可复现
> 本文档全部数字。
